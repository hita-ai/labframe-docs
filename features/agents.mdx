# Agents

An agent in LabFrame is more than a chatbot. Each agent has:

- A persistent cloud workspace (full Linux environment)
- Access to coding tools, file operations, and web browsing
- Conversation history across multiple sessions
- Configurable system prompts and model selection

## Creating an agent

Click **New Agent** in the sidebar. You can set:

| Setting | Description |
|---------|-------------|
| **Name** | A display name for your agent |
| **Icon** | An emoji identifier |
| **System prompt** | Instructions that shape how the agent behaves |
| **Model** | The AI model powering the agent (e.g., Claude, GPT) |

## Projects (conversations)

Each agent can have multiple **projects** — separate conversations with their own chat history and workspace scope. Click the **+** button next to an agent to create a new project.

Projects keep different tasks isolated while sharing the same agent workspace.

## What agents can do

Agents have access to a full set of tools:

- **Read and write files** in the workspace
- **Run shell commands** in the cloud environment
- **Browse the web** for research
- **Generate documents** — PDFs, slides, spreadsheets
- **Build and deploy websites** as `.webapp` artifacts
- **Install packages** and use any language runtime available in the environment

## Agent models

You can switch between available models using the model picker at the bottom of the chat input. Different models have different strengths — some are better at coding, others at research or creative writing.

## Sharing agents

You can invite collaborators to your agent with different access levels:

| Role | Permissions |
|------|-------------|
| **Owner** | Full control — rename, delete, manage access |
| **Editor** | Send messages, view workspace, create projects |
| **Viewer** | Read-only access to conversations and workspace |
