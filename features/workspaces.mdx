# Workspaces

Every agent's workspace is a real filesystem backed by a cloud VM. Files your agent creates persist across conversations and are accessible through the workspace browser.

## Browsing files

Switch to the **Workspace** tab in any agent view to browse the file tree. You can:

- **Single-click** a file to preview it in the right pane
- **Double-click** a `.webapp` file to open the deployed site
- **Download** any file using the download button in the toolbar
- **Upload** files by dragging them into the workspace or using the upload button

## File previews

LabFrame renders inline previews for common file types:

| Type | Preview |
|------|---------|
| Code files (`.ts`, `.py`, `.rs`, etc.) | Syntax-highlighted source view |
| Markdown (`.md`) | Rendered markdown |
| PDF (`.pdf`) | Embedded PDF viewer with zoom controls |
| Images (`.png`, `.jpg`, `.svg`, etc.) | Inline image display |
| Documents (`.docx`, `.xlsx`, `.pptx`) | Rendered document preview |
| Web apps (`.webapp`) | Metadata pane with Open App button |

## Workspace scope

When you create a **project** (conversation), the agent can scope its work to a project-specific subdirectory. This keeps different tasks organized without cluttering the root workspace.

## Persistence

Workspace files persist as long as the agent exists. Deleting an agent also deletes its workspace. Files are not shared between agents.
